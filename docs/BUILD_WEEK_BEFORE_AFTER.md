# Pico Probe before and after Build Week

## Before the submission period

The project began as an experimental multi-model mathematical-research pipeline. A researcher could arrange model calls, but the experience was still closer to prompt orchestration than a durable research environment. The early project did not yet present one coherent Pico Probe workflow in which claims, failed routes, verification results, provenance, and researcher decisions remained inspectable from project creation through the final report.

## Added during OpenAI Build Week

During Build Week, the project was meaningfully extended and consolidated into Pico Probe. The new competition experience added the typed Claim Graph, versioned Epistemic Contracts, visual pipeline compilation, named-port messages, canonical execution envelopes, assurance-aware output validation and bounded repair, branch preservation and merging, human checkpoints, Claim Passports, Research Replay, the guided investigation, public/private libraries, encrypted BYOK provider configuration, deterministic rehearsal mode, and integrated Lean, SymPy, Monte Carlo, Python experiment, Ollama, and GPT-5.6 workflows. The React interface was reworked around this research lifecycle, while backend tests and submission documentation were expanded to verify the critical paths.

## How Codex contributed

Codex served as the principal engineering collaborator throughout the Build Week extension. It inspected the inherited codebase, traced failures across the React/FastAPI boundary, translated product requirements into data models and execution contracts, implemented and connected backend and frontend features, repaired authentication and navigation failures, strengthened provider and plugin boundaries, and repeatedly reviewed tests, packaging, security, and submission readiness. Codex was especially useful for maintaining consistency across the pipeline compiler, scheduler, protocol, graph, replay, and UI rather than treating each feature as an isolated patch.

The human creator supplied the original research problem, the anti-hallucination and researcher-autonomy principles, mathematical use cases, product direction, Pico Probe identity, interface preferences, and final architectural and submission decisions. Codex accelerated and broadened the implementation; it did not replace human ownership of the idea or the scientific standards encoded by the product.
